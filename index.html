Metode Simpleks Primal

Pengertian

Metode simpleks primal adalah algoritma iteratif untuk menyelesaikan masalah pemrograman linear dengan mencari solusi dari satu titik ekstrem ke titik ekstrem lain di sepanjang tepi daerah yang layak, sehingga nilai fungsi tujuan selalu meningkat (untuk masalah maksimasi) hingga mencapai nilai optimal.

Algoritma Simpleks Primal

1
Persiapan: Ubah masalah pemrograman linear ke bentuk standar (tambahkan variabel slack, surplus, atau artifisial)
2
Tabel Simpleks Awal: Buat tabel simpleks awal dengan solusi basis yang layak
3
Periksa Optimalitas: Periksa apakah semua koefisien pada baris fungsi tujuan sudah non-negatif (untuk maksimasi)
4
Pemilihan Variabel Masuk: Pilih kolom dengan koefisien paling negatif pada baris fungsi tujuan
5
Pemilihan Variabel Keluar: Hitung rasio nilai RHS dengan koefisien kolom pivot yang positif, pilih rasio terkecil
6
Operasi Pivot: Ubah elemen pivot menjadi 1 dan semua elemen lain pada kolom pivot menjadi 0
7
Iterasi: Ulangi langkah 3-6 hingga semua koefisien pada baris fungsi tujuan non-negatif

Kondisi Optimalitas dan Penghentian

  • Untuk masalah maksimasi: Semua koefisien pada baris fungsi tujuan (Z-row) ≥ 0
  • Untuk masalah minimasi: Semua koefisien pada baris fungsi tujuan (Z-row) ≤ 0
  • Solusi tak terbatas: Jika semua elemen pada kolom pivot ≤ 0
  • Degenerasi: Jika rasio minimum sama untuk beberapa baris

Metode Simpleks Dual

Pengertian

Metode simpleks dual menyelesaikan masalah pemrograman linear ketika kondisi optimalitas terpenuhi tetapi solusi belum tentu layak. Metode ini bekerja dari solusi optimal yang tidak layak menuju solusi optimal yang layak.

Algoritma Simpleks Dual

1
Persiapan: Bentuk tabel simpleks dengan semua koefisien baris fungsi tujuan memenuhi kondisi optimalitas
2
Periksa Kelayakan: Periksa apakah semua nilai RHS non-negatif
3
Pemilihan Variabel Keluar: Pilih baris dengan nilai RHS paling negatif
4
Pemilihan Variabel Masuk: Hitung rasio koefisien baris Z dengan koefisien negatif pada baris pivot, pilih rasio terkecil secara absolut
5
Operasi Pivot: Ubah elemen pivot menjadi 1 dan semua elemen lain pada kolom pivot menjadi 0
6
Iterasi: Ulangi langkah 2-5 hingga semua nilai RHS non-negatif

Kondisi Kelayakan dan Penghentian

  • Solusi layak: Semua nilai RHS non-negatif
  • Tidak ada solusi layak: Jika semua elemen pada baris pivot ≥ 0 dan RHS < 0
  • Degenerasi: Jika rasio minimum sama untuk beberapa kolom

Dualitas dalam Pemrograman Linear

Pengertian Dualitas

Setiap masalah pemrograman linear (primal) memiliki masalah dual terkait. Keduanya memiliki hubungan penting dimana solusi optimal dari satu masalah memberikan informasi tentang solusi optimal masalah lainnya.

Maksimasi Minimasi

Contoh Transformasi

Masalah Primal

Maksimumkan: Z = 3x₁ + 2x₂

Dengan batasan:

x₁ + x₂ ≤ 4

2x₁ + x₂ ≤ 5

x₁, x₂ ≥ 0

Masalah Dual

Minimumkan: W = 4y₁ + 5y₂

Dengan batasan:

y₁ + 2y₂ ≥ 3

y₁ + y₂ ≥ 2

y₁, y₂ ≥ 0

Teorema Dualitas

  • Teorema Dualitas Lemah: Nilai fungsi tujuan primal selalu ≤ nilai fungsi tujuan dual untuk setiap solusi layak
  • Teorema Dualitas Kuat: Jika salah satu masalah memiliki solusi optimal terbatas, maka masalah lainnya juga memiliki solusi optimal terbatas, dan nilai fungsi tujuan keduanya sama
  • Teorema Kelengkapan: Jika solusi primal tak terbatas, maka dual tidak memiliki solusi layak, dan sebaliknya

Contoh Soal

Contoh 1: Metode Simpleks Primal

Maksimumkan: Z = 3x₁ + 5x₂

Dengan batasan:

x₁ + 2x₂ ≤ 8

3x₁ + 2x₂ ≤ 12

x₁, x₂ ≥ 0

Contoh 2: Metode Simpleks Dual

Maksimumkan: Z = 2x₁ + 3x₂ + 4x₃

Dengan batasan:

x₁ + x₂ + 2x₃ ≤ 10

2x₁ + 3x₂ + x₃ ≤ 15

x₁ + 2x₂ + 3x₃ ≤ 20

x₁, x₂, x₃ ≥ 0

Contoh 3: Masalah Minimasi

Minimumkan: Z = 4x₁ + 2x₂

Dengan batasan:

3x₁ + x₂ ≥ 12

2x₁ + 2x₂ ≥ 10

x₁, x₂ ≥ 0

Bantuan

Cara Menggunakan Aplikasi

  1. Pilih metode penyelesaian (Primal atau Dual)
  2. Tentukan tipe masalah (Maksimasi atau Minimasi)
  3. Masukkan jumlah variabel dan batasan
  4. Klik "Buat Form" untuk menghasilkan form input
  5. Masukkan koefisien fungsi tujuan dan batasan
  6. Pilih tipe batasan untuk setiap baris (≤, =, ≥)
  7. Klik "Selesaikan" untuk melihat langkah-langkah penyelesaian
  8. Gunakan tombol navigasi untuk melihat setiap langkah iterasi

FAQ

Kapan sebaiknya menggunakan metode simpleks primal?

Metode simpleks primal digunakan ketika:

  • Semua nilai RHS (b) non-negatif
  • Tersedia solusi basis awal yang layak
  • Masalah dalam bentuk standar (umumnya batasan ≤ untuk maksimasi atau ≥ untuk minimasi)
Kapan sebaiknya menggunakan metode simpleks dual?

Metode simpleks dual digunakan ketika:

  • Semua koefisien pada baris fungsi tujuan memenuhi kondisi optimalitas (≥ 0 untuk maksimasi)
  • Tetapi solusi belum layak (ada nilai RHS yang negatif)
  • Ketika akan melakukan analisis sensitivitas
Bagaimana menangani batasan '=' dalam simpleks?

Untuk batasan '=', kita tidak menambahkan variabel slack atau surplus, tetapi mungkin perlu menambahkan variabel artifisial untuk mendapatkan solusi basis awal yang layak. Aplikasi ini secara otomatis menangani transformasi tersebut.

Apakah aplikasi dapat menangani masalah yang tidak memiliki solusi?

Ya, aplikasi akan mendeteksi jika masalah:

  • Tidak memiliki solusi layak
  • Memiliki solusi tak terbatas
  • Memiliki solusi degenerasi

Dan akan menampilkan pesan yang sesuai.
